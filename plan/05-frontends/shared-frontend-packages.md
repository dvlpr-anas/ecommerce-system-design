# Shared Frontend Packages

## Purpose

Stop reimplementing the same TypeScript client / Tailwind config / shared types in three apps. `packages/api-client-ts` is generated from OpenAPI; `packages/ui-web` holds Tailwind + Radix primitives shared between customer-web and admin-web; `packages/events-ts` mirrors `pkg/events` for any SSE/WS consumer.

## Inputs / Prerequisites

- `pnpm-workspace.yaml` set up in [`../01-foundation/monorepo-structure.md`](../01-foundation/monorepo-structure.md)
- OpenAPI codegen pipeline working ([`openapi-codegen.md`](./openapi-codegen.md))

## Tasks

1. [ ] `packages/api-client-ts/`:
   - Generated from every service's OpenAPI spec
   - Zod schemas exported for runtime validation
   - `fetch`-based client; consumers wire auth header via interceptor
   - Internal version bump on every spec change
   — effort: M
2. [ ] `packages/ui-web/`:
   - Tailwind preset (colors, type scale, spacing)
   - Radix Primitives wrappers (Button, Dialog, Dropdown, Tabs, Toast)
   - Theme tokens (light + dark)
   - Storybook setup for visual regression tests
   — effort: L
3. [ ] `packages/events-ts/`:
   - TS types mirroring `pkg/events` Go structs
   - Generated by `task codegen:events` (Go AST → TS via a small generator, or manually maintained with CI diff guard)
   - Used by admin-web for SSE-based saga timeline (post-MVP)
   — effort: M
4. [ ] `packages/eslint-config/` + `packages/prettier-config/` shared by all three apps — effort: S
5. [ ] CI matrix: every PR builds all packages first, then consumers — effort: S
6. [ ] Publish strategy: internal-only; consumers reference `workspace:*` versions — no public npm publishing — effort: S

## Deliverables

- Three populated packages
- Storybook for `packages/ui-web` published as static site (internal)
- Shared ESLint / Prettier configs

## Exit Criteria

- [ ] All three apps import from `packages/api-client-ts` and `packages/ui-web`
- [ ] A change to an OpenAPI spec produces a regenerated client; consumers compile cleanly
- [ ] Storybook covers every Radix wrapper

## References

- Design doc: §14 Directory Structure (`packages/`)

## Risks & Open Questions

- Mobile cannot use `packages/ui-web` (React Native vs DOM components) — keep that explicit; do not try to share UI components across web and mobile.
- `events-ts` drift from `pkg/events` is a real risk — automate or CI-guard the diff.
